# 執行摘要

## 產品願景

打造一個人可以**完全信任、長期依賴、並且會自我進化**的 Life Coach AI。這不是單一AI助手，而是一組互相協作的AI Agents，圍繞職涯發展、財務規劃、技能成長、健康管理、人際影響力、決策與戰略規劃等不同人生領域運作。

## 核心差異化

| 維度 | 傳統AI助手 | 本系統 |
|------|-----------|--------|
| 架構 | 單一Agent | 多Agent協作 |
| 記憶 | 短期對話 | 三層記憶架構（短期/中期/長期） |
| 進化 | 靜態模型 | 持續自我進化 |
| 互動 | 被動問答 | 主動教練式對話 |
| 個人化 | 表面標籤 | 深度心理-行為-情境畫像 |

## 關鍵成功指標

- **用戶目標達成率**: 70% (3年目標)
- **長期留存率**: >60% (6個月)
- **用戶主動對話比例**: >40%
- **NPS評分**: >50
- **LTV/CAC**: >4x

## 1.2 Agent協作拓撲詳細設計

### 1.2.1 領域專家Agent (Domain Expert Agents)

| Agent名稱 | 核心職責 | 輸入 | 輸出 | 觸發條件 |
|-----------|----------|------|------|----------|
| **CareerAgent** | 職涯規劃、技能評估、轉職建議 | 用戶背景、職業目標、市場數據 | 職涯路徑圖、學習計劃、行動清單 | 用戶提及工作、職涯相關話題 |
| **FinanceAgent** | 財務規劃、預算管理、投資建議 | 收入支出數據、財務目標、風險偏好 | 預算方案、儲蓄計劃、投資組合 | 用戶提及金錢、財務決策 |
| **SkillAgent** | 技能學習路徑、學習方法優化 | 當前技能、目標技能、時間資源 | 學習計劃、資源推薦、進度追蹤 | 用戶想學習新技能 |
| **HealthAgent** | 身心健康、作息管理、壓力調適 | 健康數據、生活習慣、症狀描述 | 健康計劃、作息建議、預警信號 | 用戶提及健康問題或目標 |
| **RelationshipAgent** | 人際關係、溝通技巧、社交策略 | 關係情境、溝通記錄、情感狀態 | 溝通建議、關係修復策略 | 用戶提及人際困擾 |
| **DecisionAgent** | 重大決策分析、選項評估 | 決策情境、可選方案、評估標準 | 決策分析報告、推薦選項 | 用戶面臨重大選擇 |

### 1.2.2 協作模式設計

| 協作模式 | 適用場景 | 處理時間 | 流程 |
|----------|----------|----------|------|
| **單領域模式** | 單一問題 (如健身建議) | < 3秒 | 意圖識別 → 記憶檢索 → 領域Agent處理 → 知識增強 → 倫理審查 → 質量檢查 → 記憶更新 |
| **多領域模式** | 跨領域問題 (如工作壓力影響健康) | < 8秒 | 複雜度評估 → 並行領域分析 → 依賴關係分析 → 迭代協調(2-4輪) → 決策融合 → 監督審查 |
| **緊急模式** | 危機干預 | < 1秒 | 風險等級評估 → 立即響應 → 並行執行(生成回應+通知監督) → 持續監控 → 升級機制 |

## 1.3 三層記憶架構

### 1.3.1 記憶層規格

| 記憶層 | 存儲介質 | 保留期 | 核心內容 | 訪問速度 |
|--------|----------|--------|----------|----------|
| **短期記憶(STM)** | Redis | 24h | 對話上下文、會話狀態、臨時工作記憶 | O(1) |
| **中期記憶(MTM)** | PostgreSQL | 90d | 行為模式、偏好學習、週期性規律 | O(log n) |
| **長期記憶(LTM)** | MinIO/S3 | 永久 | 人生軌跡、核心價值觀、關鍵里程碑 | O(1) |
| **向量記憶** | Qdrant | 永久 | 語義檢索、相似案例匹配 | 近似O(1) |

### 1.3.2 記憶衰減機制

```
短期記憶: 硬截斷 (24h後自動清除)
中期記憶: 指數衰減 (每日衰減1%，<0.2時歸檔)
長期記憶: 記憶鞏固 (每月合併相關記憶為抽象記憶)
```

### 1.3.3 記憶檢索流程

```
查詢輸入
    │
    ▼
Step 1: 查詢分析與擴展 (提取實體、識別領域、生成嵌入)
    │
    ▼
Step 2: 並行多源檢索 (STM + MTM + 向量記憶)
    │
    ▼
Step 3: 記憶排序與過濾 (相關性35% + 時間衰減25% + 重要性20% + 訪問頻率15% + 情感強度5%)
    │
    ▼
Step 4: 記憶融合與摘要 (去重、衝突檢測、時間線重建)
    │
    ▼
Step 5: 記憶注入到Prompt
```

## 1.4 決策層設計

### 1.4.1 七階段決策流程

```
Phase 1: 輸入理解與預處理 (意圖識別、情感分析、緊急度評估)
    │
    ▼
Phase 2: 記憶檢索與上下文構建
    │
    ▼
Phase 3: 多Agent協作分析 (單領域或多領域並行)
    │
    ▼
Phase 4: 決策融合與優化 (建議聚合、置信度評估、個性化適配)
    │
    ▼
Phase 5: 監督審查與質量控制 (倫理審查、安全檢查、質量評估)
    │
    ▼
Phase 6: 可解釋性生成 (理由、來源、預期效果、風險、替代選項)
    │
    ▼
Phase 7: 反饋收集與學習
```

### 1.4.2 置信度計算模型

```python
# 五因子加權模型
confidence = (
    0.25 * consistency_score +    # Agent意見一致性
    0.25 * quality_score +        # 建議質量
    0.20 * (1 - conflict_penalty) + # 衝突程度
    0.15 * memory_support +       # 記憶支持度
    0.15 * source_reliability     # 知識來源可靠性
)
```

## 2.1 反饋迴路系統

### 2.1.1 顯性反饋收集

| 反饋時機 | 延遲時間 | 問題類型 |

### 2.1.2 隱性反饋推斷

```python
BEHAVIORAL_SIGNALS = {
    # 建議採納信號
    "immediate_action": {"weight": 1.0},   # 建議後24h內採取行動
    "repeated_action": {"weight": 1.5},    # 連續多日執行
    "goal_achievement": {"weight": 2.0},   # 達成相關目標
    
    # 建議忽視信號
    "no_response": {"weight": -0.3},        # 72h無回應

第二週: 行為模式驗證
- 每日簡短check-in
- 第一次目標設定對話
- 第一次反饋給予

持續學習:
- 每次對話結束: 溝通風格微調
- 每週回顧: 執行模式、能量週期
- 每月深度對話: 價值觀變化
- 重大事件後: 壓力反應更新
```

## 3.2 個人化互動設計

### 3.2.1 溝通風格適配

| 用戶風格 | AI適配策略 | 示例 |
|----------|------------|------|
| **直接型** | 開門見山，減少鋪陳 | 「感覺你今天能量比較低，想聊聊嗎？」 |
| **間接型** | 給予鋪墊空間 | 先問「今天過得怎麼樣？」再逐步深入 |
| **詳細型** | 提供豐富資訊 | 回應長度匹配，給予多個角度分析 |
| **簡潔型** | 精煉重點 | 使用要點式回應，控制字數 |

### 3.2.2 「被理解」的關鍵時刻

1. **記得過去的細節**: 「你上週三提到的和主管的溝通問題，後來怎麼樣了？」
2. **識別模式並指出**: 「我注意到你每次週一都會特別焦慮」
3. **適時的沉默與空間**: 不是每句話都要回
4. **用戶的語言回應**: 「感覺你現在能量很低」而非「你缺乏動力」
5. **預期用戶的需求**: 「我知道你明天有那個重要會議」

## 3.3 教練式對話模式

### 3.3.1 四階段對話策略

| 階段 | 目標 | 策略 |
|------|------|------|
| **探索** | 幫助用戶更了解自己 | 多用開放式問題、反映用戶的話 |
| **規劃** | 將洞察轉化為可行計畫 | SMART目標、探索障礙和應對策略 |
| **執行** | 支持用戶行動 | 慶祝小勝利、同理困難、提醒初衷 |
| **反思** | 從經驗中學習 | 看到成長、探索學到了什麼 |

### 3.3.2 六類提問框架

| 提問類型 | 使用時機 | 示例 |
|----------|----------|------|
| **開放式** | 探索階段 | 「你現在的感受是什麼？」 |
| **引導式** | 用戶卡住時 | 「如果沒有這個限制，你會怎麼做？」 |
| **挑戰式** | 用戶自我設限時 | 「你說你做不到，但上次類似的情況你成功了」 |
| **具體化** | 概念模糊時 | 「當你說『更好』，具體是指什麼？」 |
| **連結式** | 反思階段 | 「這個經驗和之前的那個有什麼相似？」 |
| **行動式** | 規劃階段 | 「你接下來的第一步會是什麼？」 |

### 3.3.3 給建議 vs 引導的決策框架

**給予直接建議的情況**:
- 用戶明確要求
- 涉及安全或健康
- 需要專業知識
- 時間緊迫
- 用戶已經探索過

**引導用戶自己找到答案的情況**:
- 涉及價值觀或人生選擇
- 用戶情緒激動
- 反覆出現的模式
- 用戶說「我不知道」但明顯有想法

## 3.4 主動式互動機制

### 3.4.1 主動觸發情境

| 情境類型 | 觸發信號 | 介入時機 |
|----------|----------|----------|
| **情緒低潮** | 連續負面情緒、能量詞彙 | 檢測到後2-4小時內 |
| **目標危機** | 進度嚴重落後、表達放棄 | 發現後24小時內 |
| **生活事件** | 面試、考試、搬家 | 事件前1天/後當天 |
| **成就時刻** | 目標達成、分享好消息 | 即時 |
| **沉默過久** | 超過預期時間無互動 | 根據用戶模式 |

### 3.4.2 主動關懷模板

**情緒低潮型**:
> 「嗨，我注意到你這幾天似乎心情比較沉重。不需要回覆我也沒關係，只是想讓你知道我在這裡。」

**目標支持型**:
> 「記得你說這週要完成那份報告，現在進度怎麼樣？有什麼我可以幫忙的嗎？」

**成就慶祝型**:
> 「恭喜你完成這個月的運動目標！記得你剛開始的時候還說『不知道能不能堅持』，現在你做到了！」

## 3.5 避免工具化陷阱

### 3.5.1 區別於提醒工具的設計原則

| 原則 | 錯誤示範 | 正確示範 |
|------|----------|----------|
| **永遠給予「為什麼」** | 「記得今天要去運動」 | 「這週已經完成2次了，繼續保持這個節奏！」 |
| **提供選擇而非指令** | 「你現在應該去睡覺」 | 「要不要現在休息，還是你還有事情要處理？」 |
| **連結到更大的意義** | 「完成這個任務」 | 「完成這個任務，你就離『更有條理』的目標又近了一步」 |
| **回應情緒而不只是行為** | 「你還沒完成目標」 | 「我注意到這週的進度比較慢，是遇到什麼困難了嗎？」 |
| **記得過去、連結未來** | 每次對話獨立 | 「記得你上個月說想改變這個習慣，現在已經堅持3週了」 |

### 3.5.2 健康依賴 vs 過度依賴

**健康依賴特徵**:
- 用戶在困難時會想到AI
- 用戶會主動分享成就和挑戰
- AI幫助用戶變得更獨立
- 用戶仍然保持其他社交連結

**過度依賴警訊**:
- 用戶所有決定都依賴AI
- 用戶減少了與真人的互動
- 用戶對AI產生不現實的期待
- 用戶無法在沒有AI的情況下做決定
